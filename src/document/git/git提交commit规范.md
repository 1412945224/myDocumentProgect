# commit 提交规范

|   类型   |                        描述                        |
| :------: | :------------------------------------------------: |
|  build   | 编译相关的修改，例如发布版本，对项目构建或者依赖的 |
|  chore   |    其他修改，比如修改构建流程、增加依赖库、工具    |
|    ci    |                    持续集成修改                    |
|   docs   |                      文档修改                      |
|   feat   |                   新特性、新功能                   |
|   fix    |                      修改 bug                      |
|   perf   |              优化相关，比如性能、体验              |
| refactor |                      代码重构                      |
|  revert  |                   回滚上一个版本                   |
|  style   |                    代码格式修改                    |
|   test   |                    测试用例修改                    |
